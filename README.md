# Pure Coding — Code Without AI (30 Days)

> The course that fixes the "I go blank without AI" feeling. In 30 days you'll understand how computers and code actually work, and you'll write real programs **from your own head** — no AI, no copy-paste, just you and the problem.

> 🟢 **New here? Read [`START_HERE.md`](./START_HERE.md), then [`RULES_NO_AI.md`](./RULES_NO_AI.md).**

---

## Why This Course Exists

You said it yourself: *"I can't code a single line without AI. I feel blank."* That's not a talent problem — it's a **training** problem. AI did your thinking, so your thinking muscle never grew. This course grows it. By Day 30 you'll trust your own brain to solve problems.

## The Goal

- **Understand the machine:** what happens inside a computer when you write code, what the web and internet are, encoding, compilers vs interpreters, IDEs, and environments.
- **Build a thinking brain:** a repeatable problem-solving framework you apply to any problem.
- **Code from scratch:** write working programs in JavaScript with **no AI assistance.**
- **Become versatile:** read errors, debug by hand, decompose problems, and learn how to learn.

> **Language: JavaScript.** Why? It's the language your main journey (No-Plan-B) uses, so every hour here makes that easier. You only need a browser to start.

## The One Sacred Rule

🚫 **No AI in this course. None.** No ChatGPT, no Copilot, no "just ask it." That's the entire point — see [`RULES_NO_AI.md`](./RULES_NO_AI.md). Allowed: official docs, your own notes, and your own brain.

## Course Map (30 Days)

| Module | Days | What you build |
|---|---|---|
| **0 — Foundations** | 1-5 | How computers run code, encoding, compiler vs interpreter vs IDE, environments, the terminal, the internet & the web |
| **1 — Programming Logic** | 6-12 | Variables, types, operators, conditionals, loops — the atoms of all code |
| **2 — Functions & Decomposition** | 13-17 | Breaking big problems into small functions |
| **3 — Data Structures & Algorithms** | 18-24 | Arrays, strings, objects; searching, sorting, and Big-O intuition |
| **4 — Problem-Solving Mastery** | 25-30 | Daily challenges + a from-scratch capstone program |

## How It Works (Daily Loop)

1. Open the current module guide (start: `Module_0_Foundations/Module_0_Guide.md`).
2. Open today's log in that module's `Logs/` folder.
3. **LEARN** the concept → **DO** the exercises by hand/typed yourself → pass the **Transition Condition**.
4. Write your practice code in the module's `Exercises/` folder.
5. Struggle is expected. Sit in it. That struggle is the muscle growing.
